## 1. 项目背景与起源

### 1.1 缘起
作者给自己设了一个高额奖励 + 高难度目标：**如果某天能学习/工作满 12 小时，就达成目标、领取奖励**（也保留 14 小时档，14h 同时是系统上限）。为了给自己施加适度压力、并真实地量化"我今天到底专注了多久"，需要一个随手可见、随手可控的计时工具。

### 1.2 核心痛点
1. **容易分心**：需要一旦做别的事就能"一键暂停"，回来再"一键继续"，确保计入的都是纯专注时间。
2. **要随时可见**：像右下角时钟一样常驻，但更大、在屏幕上方，任何页面之上都能瞄一眼。
3. **要有正反馈**：长时间专注枯燥，需要每达成一个小时就有即时的可视化奖励来支撑动力。
4. **要能复盘**：知道自己一天专注了多久、分心了几次、每段专注多长。

### 1.3 关键设计哲学：自驱式计时（人工门禁）
计时由用户**主动开启**——只有真正进入专注状态才点开始。这带来一个重要推论：**计时期间的一切行为都默认是"有效产出"**。因此：
- 在微信/企业微信 = 一定是工作沟通，不是闲聊；
- 在看视频 = 一定是学习/工作相关内容；
- 归到"其他" = 一定是别的正经工作（写作、设计等）。

这使后续的"内容自动分类"统计天然可信，无需复杂的反作弊。

### 1.4 适度克制：上限即劝退
专注是好事，但"无止境地刷时长"反而违背初衷。产品把**有效专注上限锁在 14 小时**：徽章最多 14 枚、满 14h 后进入"劝退态"——计时数字仍走、但不再计入有效专注，并直接建议休息。工具鼓励专注，也负责喊停。

## 2. 产品目标与核心价值

| 目标 | 说明 |
|---|---|
| 量化有效专注 | 精确记录"纯专注时长"（暂停不计），以天为单位，封顶 14h |
| 目标驱动 | 自定义每日目标（默认 12h，上限 14h），达成即触发奖励仪式感 |
| 即时正反馈 | 每满 1 小时给声音 + 动画 + 徽章 + 通知的多重奖励 |
| 行为复盘 | 时间线、分心次数、最长连续专注、内容分类占比、当日多段记录 |
| 低摩擦常驻 | 永远置顶、整窗可拖、快捷键（可自定义）、可拉伸、全屏专注模式 |

## 3. 目标用户与使用场景

- **用户画像**：自由职业 / 自我驱动型学习者，一天中混合"看网课、与客户沟通、做具体工作"三类任务，需要外部工具帮自己保持专注并量化产出。
- **典型一天**：早上双击桌面图标启动 → 进入专注就点"开始" → 分心/休息就暂停 → 整点收到奖励反馈 → 中途若改变主意可"清除今日记录、从头再来"（旧记录归档保存）→ 一天结束打开复盘页回顾。
- **多屏场景**：把一块屏幕切到"全屏专注模式"当沉浸背景（极简巨字时间 + 徽章 + 进度条）。

## 4. 功能需求详述

> 截至 2026-06-15，下述功能均 ✅ 已打包进运行中的 exe。🔭 = 规划中/未做。

### 4.1 悬浮窗与计时核心 ✅
- 无边框、**永远置顶**（盖住全屏应用）、不在任务栏显示、默认屏幕顶部居中。
- 中央大号显示**今日专注总时长**（已扣除暂停），字号随窗口尺寸自适应（`clamp(40px,17vmin,120px)`）。
- **去掉了卡片边框与投影**：干净无边无晕；专注/暂停的反馈改由眼睛图标、状态文字、按钮配色承担。
- **整窗可拖**：在悬浮窗任意空白处按住即可拖动（不再限于顶部一行）；按钮/工具/弹层为 no-drag，可正常点击。

### 4.2 窗口可拉伸 + 自适应 ✅
- 窗口**可自由拉伸**：`minWidth 360`（保证 14 徽章单行）/ `minHeight 240`；上限 `maxWidth 720 / maxHeight 600` 且不超过显示器工作区约 70%——拉伸态始终明显小于全屏，两种形态界限清晰。
- 字号、徽章、浓缩条随尺寸自适应（`vmin` 基准，兼容超大屏/带鱼屏）。
- 固定初始高度 + flex 弹性布局；用户手动拉伸后不再自动改高。

### 4.3 暂停 / 继续 ✅
三种触发方式（**点中央数字不会再误触**）：
1. **大按钮**（主入口）。
2. **全局快捷键**：任何软件里都能切换；默认 `Ctrl+Alt+S`（带自动避让），**可在设置里录制自定义**。
3. 窗口聚焦时按 **Enter**。
- 暂停时计时停止、时间不计入；后台记录每次暂停的时间点。

### 4.4 徽章体系：上限 / 超额 / 劝退 ✅
- **徽章墙**：数量 = `min(14, max(目标, 已完成小时))`，**永远单行、最多 14 枚、绝不换两行**。
- **整点点亮**：专注每满 1 小时点亮一枚；单段连续专注 ≥2h 为**金色**。
- **超额福利徽章**：完成数超过"自己设的目标"但仍 ≤14 的那些，用**鎏金 + 星光发光描边**区分，表示"比计划多努力"。
- **满 14 小时 → 劝退态**：计时数字继续走，但**不再计入有效专注、不再点亮徽章**（有效专注封顶 14h）；眼睛闭上；暂停/开始按钮**整块替换成劝退提示卡**（"今天已经可以了 / 今天已经做了很多了；再做下去，很难保证你真的在专注"），不再可切换。

### 4.5 整点奖励 ✅
专注总时长每跨过 1 小时整点，同时触发：
- 🔔 **「叮~」提示音**（Web Audio 合成；达成目标用更欢快的三连音）；
- 🎆 **撒花动画**；
- **点亮一枚徽章**（金 / 鎏金规则见 4.4）；
- 🪟 **Windows 系统通知**（静音，避免与叮声重叠）；
- 满目标时额外播放**全屏庆祝**。

### 4.6 当前小时浓缩时间轴 ✅
紧贴暂停按钮上方、与按钮等长，表达**当前这 1 小时的 60 分钟纯专注**：
- **进度填充** + **内容分段着色**（看视频 / 客户沟通 / 其他工作）；
- **旗子**：每次暂停在对应位置插**自绘实心三角旗**（暂停=奶白、60 分钟终点=红）；旗子以旗杆为锚点向右展开并夹紧，**不会戳出浓缩条圆角**；
- **防密集**：两次暂停之间有效专注 <60 秒不插新旗，且每小时最多 12 面（防短期狂点把浓缩条插满）；
- 满 1 小时后重置，进入下一小时。
- 🔭 未来做成"可爱进度"：地面 + 小鹿 / 履带 + 货物 / 土地 + 萝卜 等拟物化样式。

### 4.7 专注 / 休息 / 引导 三态显示 ✅
- **专注中**：中央显示专注总时长（大、居中）；状态为睁眼 +「专注中」。
- **暂停休息**：专注总时长缩到左上角（"已暂停"之后的小字）；中央换成**休息计时**（从 0 起算）；状态为闭眼 +「已暂停」。
- **引导/未开始**：一次都还没记录时，中央显示 `00:00:00` 作视觉引导，告诉你"还没开始"。
- ⚠️ 三态之间为**瞬时切换、无动效**（早期做过"数字飞行/休息滑入滑出"，按作者要求**已去掉**以求简洁）。

### 4.8 全屏专注模式（极简巨字）✅
工具栏全屏按钮 → 撑满整块屏幕的沉浸背景，三段式布局：
- **顶部一行**：徽章（随屏放大）；
- **中部居中**：超大专注时间（`vmin` 基准、带鱼屏不溢出）+ 一行文案 + **低调无边框的暂停图标**（信息层级低、不抢视觉，仍可点）；
- **最底边**：浓缩条贴底 + 三角旗立其上（"地面插旗"观感）。
- 角落有退出全屏按钮；与"可拉伸的悬浮窗"是两套独立形态。

### 4.9 内容自动分类 ✅
计时运行时，每 5 秒读取**当前最前台窗口**（进程路径 + 标题），归类并累计：
- **客户沟通**：前台为企业微信 / 微信 / QQ（按进程名精确识别）；
- **看视频/网课**：前台为浏览器且标题命中视频/网课关键词；
- **其他工作/学习**：其余。
- 悬浮窗底部实时显示「当前：xx」；复盘页展示三类时长 + 占比条。
- 全自动、不手动纠正；规则集中在 `src/categories.js`，便于扩充。

### 4.10 设置（独立窗口）✅
设置从"悬浮窗内的蒙版弹层"改为**独立窗口**（`src/settings.html`，像复盘窗那样），不受悬浮窗尺寸限制；**打开设置即让计时进入暂停**（设置时也是休息）。内含三块：
- **今日目标时长**：预设 12h / 14h + 自定义；输入 >14 时给**文字提示 + 窗口震动 + 提示音**（即使静音也能靠震动察觉）并修正为 14；保存后即时同步悬浮窗徽章。
- **开机自启**开关。
- **快捷键**：一行展示当前键 +「修改」按钮；点修改后**直接按下想要的组合键即录制保存**（需含 Ctrl/Alt/Shift，Esc 取消，冲突会提示重设），存于 `settings.json`。

### 4.11 清除今日记录（归档，不真删）✅
设置窗底部「清除今日记录，从头开始」（低调描边 + 二次确认）。用于"上午做了一些、后来改主意想重新计"的场景：
- 点击后**当前记录归档保存**进 `data.archives`，悬浮窗从 0 重新计；
- 归档的旧记录在**今日复盘**底部「今日早先的记录（已清除归档）」区仍可查看（每段的时长 + 起止时间）——相当于一天可起多份专注。

### 4.12 数据记录与复盘 ✅
- 按天一个 JSON，状态每变即写盘 + 每 10 秒自动保存（防崩溃）。
- **复盘页**（独立窗口）：当天时间线、每段专注明细、最长连续专注（高亮）、暂停次数、目标完成度、内容分类占比、**今日早先的归档记录**。
- 复盘窗**无英文菜单栏**、标题「溯迹」统一橙、**任务栏图标与软件 logo 一致**。

### 4.13 视觉与图标 ✅
- **南瓜暖秋色卡**（CSS 变量）：BISTRE 深棕底 / COCOA 主橙 / PERSIAN 次橙 / NAVAJO 奶白 / KHAKI 暖灰；暂停态暖灰。
- **自绘眼睛状态图标**：睁眼=品牌放射眼（杏仁眶+虹膜环+瞳孔+上方光芒）/ 闭眼=轮廓线+下睫毛（常量在 `src/timer.js` 的 `EYE_OPEN`/`EYE_CLOSED`）。
- 图标库 **Lucide**（vendoring 进 `src/lucide.min.js`）用于工具栏/大按钮/复盘卡等；三角旗因 lucide 无合适款式而自绘。**约定：以后加图标先在 lucide 找，没有合适的或不满意才自绘。**

### 4.14 分发与系统集成 ✅
- 打包为双击即开的 `.exe`（输出 `dist\溯迹-win32-x64\溯迹.exe`，眼睛图标已烧入）；桌面快捷方式指向它。
- **开机自启**：经查系统层面当前为关闭状态（注册表/启动文件夹/任务计划均无自启项），可在设置里开关。

## 6. 数据模型（按天 JSON）

```jsonc
{
  "date": "2026-06-15",
  "goalHours": 12,
  "totalFocusedMs": 0,                 // 今日专注总时长（毫秒，有效专注封顶 14h）
  "sessions": [                        // 每段连续专注（继续→暂停）
    { "start": "ISO", "end": "ISO", "durationMs": 0 }
  ],
  "pauses": [                          // 每段暂停（暂停→继续）
    { "pauseAt": "ISO", "resumeAt": "ISO|null" }
  ],
  "longestStreakMs": 0,               // 最长单段连续专注
  "categories": { "video": 0, "client": 0, "other": 0 }, // 内容分类累计(ms)
  "currentHour": {                     // 当前小时浓缩条
    "hourIdx": 0,
    "samples": [],                     // 按序的分类串，每个代表 5 秒
    "pauseMarks": []                   // 本小时内暂停发生的"已专注ms"位置
  },
  "archives": [                        // 被"清除"归档的早先记录（复盘仍可见）
    {
      "totalFocusedMs": 0, "sessions": [], "pauses": [],
      "longestStreakMs": 0, "categories": {}, "archivedAt": "ISO"
    }
  ]
}
```
- 持久化路径：`%APPDATA%\study-timer\data\YYYY-MM-DD.json`（与应用显示名解耦，改名不丢数据）。
- **应用设置**（自定义快捷键等）单独存 `%APPDATA%\study-timer\settings.json`。
- 旧文件缺字段自动补默认值（向后兼容，含 `categories`/`currentHour`/`archives`）。

## 7. 技术架构

- **框架**：Electron（主进程 `main.js` + 预加载桥 `preload.js` + 渲染层 `src/`）。
- **窗口**：三个 `BrowserWindow`——悬浮窗（frame:false / transparent / alwaysOnTop('screen-saver') / skipTaskbar / 可拉伸带 min·max）、复盘窗、设置窗；全屏由主进程 `setBounds` 撑满当前屏 + 渲染层切 `body.fullscreen`。
- **全局快捷键**：`globalShortcut` 注册；用户自定义优先（`settings.json`），否则候选列表逐个避让；`set-hotkey` IPC 动态改注册。
- **前台窗口检测**：`get-windows`（ESM，主进程动态 import）。
- **奖励音效 / 超限提示音**：Web Audio 实时合成（无音频文件依赖）。
- **真值唯一**：渲染层持有 `data` 并负责累计/保存；主进程做读写、系统能力（通知/快捷键/检测/窗口）、以及跨窗同步（设置窗改目标/清除 → `set-goal`/`clear-today` IPC → 通知悬浮窗 `goal-updated`/`day-cleared`）。
- ⚠️ **前端库一律 vendoring 进 `src/`**：electron-packager 会裁掉包的 `dist/` 子目录，导致打包后 404 卡死（lucide 已放 `src/lucide.min.js`，`timer.js` 有兜底）。

```
└── PENDING.md           改动历史 / 重新打包命令
```

## 8. 非功能性需求

- **隐私**：所有数据仅本地存储；前台窗口检测只读进程名与标题，不上传、不记录截图。
- **健壮**：检测/音频/通知/图标全程 try-catch，失败不影响计时主流程；10 秒自动保存防崩溃丢数据；"清除"为归档而非删除，不丢数据。
- **低打扰**：通知静音；悬浮窗不进任务栏；暂停态视觉收敛（暖灰）；满 14h 主动劝退。
- **节制**：有效专注上限 14h，超限不再计入。
