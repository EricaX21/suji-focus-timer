## 为什么会有溯迹

市面上成熟的番茄钟，几乎都建立在同一个假设上：**你能在规定的时间里专注，也能在规定的时间里休息。**

可这个假设并不适用于所有人。对一部分人——尤其是注意力天生更跳跃、容易分神、节奏难以被外部规则框住的人（ADHD 倾向者往往如此）——「25 分钟工作 + 5 分钟休息」这种刚性结构反而成了负担：状态来的时候被铃声打断，状态没来的时候又被计划逼迫；休息从来不肯准点发生；一旦某天没按计划执行，挫败感就会让整套系统崩塌。

**溯迹选择了相反的思路：不去规定专注，而是如实地度量专注。**

你只在「真正进入状态」的那一刻才点开始；它不催你、不打断你，只安静地把你**真实产出的时间**累积、量化、可视化。它默认你会分神、会不规律地休息、会有状态起伏——并把这些都接纳进设计里，再用奖励与成就感温柔地把你往前推。

> 它不是要把你改造成一个能被计划管理的人，而是为「本来就不那么好被计划管理」的你，做一个真正合身的工具。

## 核心理念，落到四个设计上

**① 自驱式计时 —— 只度量真实专注**
计时由你主动发起，只在专注时走表。量化的是「今天到底有效产出了多久」，而不是「你在桌前坐了多久」。每满一小时点亮一枚徽章，把抽象的坚持变成看得见的累积。

**② 接纳分神 —— 自动暂停 + 时间裁决**
连续数分钟没有键鼠操作时，溯迹会**自动暂停并把计时回拨到你真正走神的那一刻**，再回头问你：刚才是在专注（看书 / 纸质资料 / 手机），还是分神了？——选前者，时间无缝还给你；选后者，那段就不计入。专注的纯度，由你自己定义，而不是被一刀切。

**③ 不规律的休息，也被照顾**
连续专注满两小时会温柔提醒你起身、喝水、远眺；想休息随时停，不必等某个固定刻度。休息不是被安排的，而是被允许的。

**④ 鼓励，而非惩罚**
多天计划用「完成度按实际小时占比」如实反映进度——漏了一天不会被打回重来，只是诚实地降一点、并给你「延后一天」之类的补救余地。目标达成有撒花、有提示音、有可自定义的奖励牵引。它的每一处反馈，都是为了让你想继续，而不是让你想放弃。

## 功能一览

- 🪟 **置顶悬浮窗**：可拖动、可拉伸、可全屏极简巨字；轻量陪伴不抢戏
- 🎯 **目标 / 多天计划 / 奖励**：设定每日目标小时、维持天数与兑现奖励，计划按奖励自动取有趣的名字
- 🧠 **内容自动识别**：感知你正在用的软件/网站，把时间分成「看视频 / 沟通 / 工作学习」并细分到具体应用
- ⏸️ **分神自动暂停 + 时间回拨**：如上，专注纯度由你裁决
- 🌙 **熬夜党模式**：把「一天」的界线挪到凌晨，深夜的专注仍归当天
- 📺 **全屏看视频自适应**：检测到全屏播放时，悬浮窗自动缩成顶部一个迷你数字，或完全隐藏
- 📊 **复盘看板**：每小时产出折线图、内容分类可展开、归档记录可查
- 🏅 **仪式感**：整点奖励、达成庆祝、收工总结独立成窗
- 🔒 **纯本地**：所有数据只存在你自己的电脑上，不联网、不上传

## 技术栈与实现

- **Electron** + **原生 HTML/CSS/JavaScript**（无前端框架，零运行时依赖负担）
- 多窗口架构：悬浮窗 / 设定窗 / 设置窗 / 复盘窗 / 今日总结窗，主进程统一调度
- 分神检测基于 `powerMonitor` 系统空闲时间，**无需全局键鼠钩子**，看视频时自动豁免
- 前台应用识别用 `get-windows`，外部全屏检测复用同一来源比对屏幕尺寸
- 数据按日存为本地 JSON，跨天计划与设置独立持久化，应用名与数据解耦（改名不丢数据）
- 自绘 SVG 眼睛状态图标 / 三角小旗 / 每小时折线图，配套一套南瓜暖秋设计令牌（见 `DESIGN.md`）
